# Flutter 项目基础功能框架

这是一套完整的 Flutter 项目标准化方案，旨在帮助团队快速启动新项目，复用已有功能和 AI 开发技能。

## 📚 文档结构

### 核心文档

1. **[Flutter 项目基础功能框架](docs/flutter-framework.md)**
   - 定义了 4 个开发阶段（初始化、MVP、生产就绪、高级特性）
   - 每个阶段的必需功能清单
   - 30+ 个可复用的 AI 技能建议

2. **[Flutter 功能支持状态矩阵](docs/feature-status.md)**
   - 35+ 个功能点的状态跟踪
   - 按优先级分类（P0/P1/P2/P3）
   - 技能开发路线图（Q3 2026 - 2027 H1）

3. **[Flutter 技能库实施路线图](docs/implementation-roadmap.md)**
   - 5 个实施阶段的详细计划
   - 技能开发流程和质量标准
   - 关键指标和风险缓解策略

## 🎯 项目目标

- **标准化**: 统一的项目结构和开发规范
- **快速启动**: 新项目 30 分钟内完成基础搭建
- **知识复用**: 将团队经验沉淀为可复用的 AI 技能
- **质量保证**: 内置最佳实践和质量检查

## 🚀 快速开始

### 项目生命周期阶段

#### 阶段 0：项目初始化（Project Bootstrap）
- 项目结构标准化
- 依赖管理配置
- 代码规范配置
- 多环境配置
- CI/CD 基础

#### 阶段 1：MVP 核心功能（MVP Core）
- 状态管理（推荐 Riverpod）
- 路由导航（推荐 go_router）
- 网络请求（推荐 Dio）
- 本地存储
- UI 主题系统
- 错误处理

#### 阶段 2：生产就绪（Production Ready）
- JWT/OAuth 认证
- 安全性加固
- 日志系统（Crashlytics）
- 性能监控
- 分析统计
- 国际化支持
- 测试覆盖

#### 阶段 3：高级特性（Advanced Features）
- 离线优先架构
- 推送通知
- 深链接
- 原生桥接
- 高级动画
- 图片处理
- 地图集成
- 支付集成

## 📊 技能开发优先级

- **P0**: 必需功能，所有项目都需要
- **P1**: 重要功能，大多数项目需要
- **P2**: 可选功能，根据项目需求选择
- **P3**: 高级特性，特定场景使用

## 🛠️ 技术决策

- **状态管理**: Riverpod
- **路由**: go_router
- **网络请求**: Dio
- **所有技能遵循**: 模块化、可组合、文档化原则
